## Статус проекта
Backend: **13 реализованных модулей** (9 базовых: 7 core + Admin + Website Builder, offline-сьют 455/455; + Спринт 1 Figma-gap: Chats/Profile/Preferences; + **Token Purchase (MVP)** — consumable IAP, без миграции, общий offline-сьют **681/681 зелёный**) + observability. Инфра: Dockerfile, docker-compose + e2e-override, observability-стек. **Deploy-топология (2026-06-02, [ADR-017](adr/ADR-017-shared-server-traefik-deploy.md)): общий сервер за внешним Traefik + GitHub Actions SSH** (`appleboy/ssh-action`, `script_stop: false`, remote-скрипт под `set -uo pipefail` без `-e`; per-instance loop по `$INSTANCES`: `git pull --ff-only` → explicit `build api migrate` → `run --rm migrate` → `up -d --no-build` → readiness-gate `${proj}-api-1` healthy → NON-FATAL smoke `/healthz`; реальные сбои копятся в `$FAILED` → финальный `exit 1` краснит job, см. [07-deployment.md §Процедура деплоя](07-deployment.md#процедура-деплоя-github-actions--ssh)). Выкатка — **gated `deploy`-job в `ci.yml`** (`needs: [quality, test, build-image]`, `if: ref==main && success()`) — deploy только после зелёного CI на `main`; `deploy.yml` — ручной (`workflow_dispatch`). Prod-артефакты: `docker-compose.prod.yml` (Traefik-labels, сеть `web` external, `expose: 8000` без портов 80/443), `.env.prod.example`, `.github/workflows/ci.yml` (+gated deploy) и `deploy.yml` (ручной) — см. [07-deployment.md](07-deployment.md#cicd-gate). Прежние Caddy/`deploy-vps.sh` — legacy (TLS/reverse-proxy = внешний Traefik). **Развёрнут на `broadnova.shop`** (контейнеры healthy, миграции `0001`→`0005` применены; staging-режимы `DOCS_ENABLED=true`/`STOREKIT_TEST_MODE=true`; **JWT issuer — встроенный в backend, реализован, [Q-005-1](99-open-questions.md) Closed ([ADR-018](adr/ADR-018-embedded-auth-issuer.md))**, prod-требование — сгенерировать и сконфигурировать RSA-пару подписи, иначе `/v1/auth/*` → `503`). Миграции — цепочка `0001`→`0002`→`0003`→`0004`→`0005`→`0006`→`0007` (expand-only), применяются на pre-deploy: `0003` — `projects`/`site_files` website-builder; `0004` — Figma-gap Спринт 1 (`chat_sessions.title`/`assistant_mode`/`is_pinned`, `users.display_name`, таблица `user_preferences`, расширение enum `byok_key_status`); `0005` — auth-issuer (`auth_devices`, `auth_refresh_tokens`); `0006` — `chat_steps.seq` (монотонный порядок реконструкции, backfill по `(created_at,id)`, индекс `ix_steps_session_seq`; [ADR-021](adr/ADR-021-deterministic-step-order-and-block-normalization.md), BUG-5); `0007` — `chat_sessions.project_id` → nullable (опциональный projectId, «чистый чат» без website-builder; [ADR-022](adr/ADR-022-optional-project-and-tool-gating.md)).

**Расширение (2026-06-02, РЕАЛИЗОВАНО — auth + tools catalog):** две новые фичи к реализованному backend; **код реализован, отревьюен и протестирован** (offline-сьют **775/775 зелёный**, production-ready), Swagger обновлён.
1. **Встроенный auth-issuer** (модуль [auth](modules/auth/README.md), [ADR-018](adr/ADR-018-embedded-auth-issuer.md), **закрывает [Q-005-1](99-open-questions.md) реализацией**): backend САМ выпускает (`POST /v1/auth/register|token|refresh`, `GET /v1/auth/jwks`) и верифицирует RS256 JWT (self-consistent, `iss=https://broadnova.shop`, `aud=claude-ios`); первичная аутентификация — **device-based** (анонимная, `deviceId`→`userId`); access-token TTL 1ч + opaque refresh-token (30д, single-use rotation, hashed-store). Эндпоинты `/v1/auth/*` — public (без `Authorization`), защита per-IP rate-limit; коды `200`/`422`/`429`/`503` (`503` — issuer не сконфигурирован, нет приватного ключа). Provisioning `users` явный при register, lazy-fallback ([ADR-007](adr/ADR-007-lazy-user-provisioning.md)) сохранён; trial/policy не сломаны. Ключи: `JWT_PRIVATE_KEY`/`JWT_PUBLIC_KEY` (с `\n`-экранированием) ИЛИ `*_PATH` (файл), приоритет у файла; приватный ключ — секрет под redaction. Новые таблицы `auth_devices`/`auth_refresh_tokens` (миграция **`0005`**, цепочка `0001`→`0005`). Email/пароль и Apple Sign-In — опциональное расширение ([Q-018-2](99-open-questions.md)), путь не закрыт.
2. **Каталог инструментов** `GET /v1/tools` ([ADR-019](adr/ADR-019-tools-catalog-endpoint.md), [chat-orchestrator/02-api-contracts](modules/chat-orchestrator/02-api-contracts.md#get-v1tools--каталог-инструментов-adr-019)): **реализован**, JWT-protected (`bearerAuth`), возвращает все 13 tools (`name` доменное, `description`, `mutating`, `execution` client/server, `inputSchema`); источник — `src/app/chat/tools.py` (single source of truth). Существующее (биллинг/policy/provisioning/tool-loop/admin/preview/website-builder/token-purchase/`JwtVerifier`) **не ломается**.

**Расширение (2026-06-01, реализовано):** добавлены и реализованы два модуля — **Admin** (начисление кредитов под изолированным `X-Admin-Token`, [ADR-009](adr/ADR-009-admin-token-auth.md): `POST /v1/admin/wallet/grant`, `GET /v1/admin/wallet/{userId}`) и **Website Builder** (хранение сгенерированных сайтов `projects`/`site_files` — миграция `0003`, server-side tools `site.*` [ADR-011](adr/ADR-011-server-side-tools.md), backend-hosted preview по signed URL `GET /v1/preview/{projectId}/{token}/{path}` [ADR-010](adr/ADR-010-backend-hosted-preview.md)). Контракты, data model, threat model и фазы зафиксированы; **код реализован, отревьюен и протестирован** (offline-сьют 455/455 зелёный, включая e2e admin-grant/get-wallet и preview write→signed-URL→serve). Деплой/публикация сайтов в интернет — отложен (вне scope этого прохода). Хранение контента сайтов в БД (BYTEA) на старте → миграция в object-storage зафиксирована как [TD-009](100-known-tech-debt.md). Сводный API-справочник для PM/интеграторов — [API-REFERENCE.md](API-REFERENCE.md).
**Live-подтверждение website-builder:** offline e2e (admin/preview) зелёные; прогон live website-builder с реальным Claude (server-side `site.*` tool-loop end-to-end) ожидает эмпирического подтверждения после пополнения баланса Anthropic-аккаунта — это **внешнее ограничение оплаты, не дефект кода**; контракты и offline-проверки полностью покрывают поведение.
**E2E (2026-05-25): пройден полностью** — весь [09-e2e-testing.md §4](09-e2e-testing.md#4-e2e-сценарии-acceptance-для-прогона) зелёный против живого сервиса (Claude — реальный Anthropic, StoreKit — HS256 test-mode), DoD §5 закрыт, `production_ready=true`, расхождений docs↔поведение нет.
