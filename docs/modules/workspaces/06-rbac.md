# Workspaces — RBAC

Реализация — [ADR-036](../../adr/ADR-036-workspaces-implementation.md).

- Роль `user`. Все операции (CRUD workspace, файлы-знания, привязка чатов) ограничены ресурсами `sub`.
- Чужой/несуществующий workspace → `404` (никогда не раскрывать чужое существование).
- Чужой/несуществующий `fileId` при DELETE → `404`.
- Привязка чата к чужому/несуществующему workspace (`workspaceProjectId` в `/chat/run`) → `404 workspace_not_found` при создании сессии.
- Нет admin-операций.
- `instructions` и тексты файлов — пользовательский контент, подаётся модели как контекст/промт (не исполняется). Логирование тела запросов/контента файлов — под общими redaction-правилами ([05-security.md §Логирование](../../05-security.md#логирование-безопасное)): содержимое `instructions`/файлов не пишется в логи.
- Модель угроз инъекции чужого контента — [05-security.md §Workspaces](../../05-security.md).
