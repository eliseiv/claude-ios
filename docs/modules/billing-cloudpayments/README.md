# Module: Billing — CloudPayments (broadapps / YooKassa, RU-путь)

- Статус: Спроектирован, ожидает реализации
- Ответственность: **RU-путь биллинга** (broadapps `pay.broadapps.dev`, фронтит YooKassa), независимый от Adapty и StoreKit. Две половины:
  - **Исходящая** — `POST /v1/billing/cloudpayments/checkout` ([ADR-051](../../adr/ADR-051-cloudpayments-checkout-payment-link.md)): наш JWT-эндпоинт создаёт платёжную ссылку через broadapps `/payments/link`; `userId` из JWT (не из тела) → фикс «потерянных платежей».
  - **Входящая** — `POST /v1/billing/cloudpayments/webhook` ([ADR-050](../../adr/ADR-050-cloudpayments-webhook.md)): колбэк broadapps в формате CloudPayments; идемпотентная активация подписки / начисление token-пакета и грант кредитов.

## DoD — Checkout ([ADR-051](../../adr/ADR-051-cloudpayments-checkout-payment-link.md))
- `POST /v1/billing/cloudpayments/checkout` — JWT (`CurrentUser`); нет/невалидный → `401`. **`userId` из JWT `sub`, НЕ из тела.**
- Тело (StrictModel): `productId` (валидируется `classify_product`-allowlist; unknown/некредитуемый → `422`), `customerEmail` (`EmailStr`; невалидный → `422`). Лишние поля → `422`.
- Исходящий httpx `POST {CLOUDPAYMENTS_API_BASE}/payments/link`, **multipart** {`app_id`,`product_id`,`user_id`(=sub),`customer_email`}, `Authorization: Bearer <CLOUDPAYMENTS_API_TOKEN>`, таймаут 15с.
- Ошибки httpx/не-2xx/malformed → `502 upstream_error` (без утечки токена/деталей). Успех → `200` {`paymentId`,`paymentUrl`,`status`,`expiresAt`}.
- `CLOUDPAYMENTS_APP_ID`/`CLOUDPAYMENTS_API_TOKEN` не заданы → `503` (только avelyra). Rate-limit → `429`.
- **Без миграции.** `customer_email`/`CLOUDPAYMENTS_API_TOKEN`/`app_id` не логируются/не персистятся; лог `cloudpayments_checkout_outcome` (allowlist). Upstream фиксирован (нет SSRF).
- Swagger-чистота: user-facing строки без ADR/TD/Q.

## DoD — Webhook ([ADR-050](../../adr/ADR-050-cloudpayments-webhook.md) → **АКТУАЛЬНО [ADR-054](../../adr/ADR-054-cloudpayments-webhook-payment-verification.md)**)

**[ADR-054](../../adr/ADR-054-cloudpayments-webhook-payment-verification.md) — публичный вебхук + верификация (текущий контракт):**
- **Эндпоинт публичный: `401` НЕ выдаётся.** broadapps шлёт колбэк **без авторизации** (`authScheme=none`) и без подписи. `require_cloudpayments_webhook` — наблюдательная non-blocking (лог `cloudpayments_webhook_auth_observed`, DEBUG/INFO). **Гейт активации — `CLOUDPAYMENTS_API_TOKEN`** (пуст → `500` misconfigured ⇒ только avelyra); `CLOUDPAYMENTS_WEBHOOK_TOKEN` — легаси/опционален.
- **Per-IP rate-limit** `enforce_cloudpayments_webhook_limits` (дефолт `CLOUDPAYMENTS_WEBHOOK_RATE_LIMIT_PER_IP=120`/мин) → `429`.
- Парсинг колбэка: обязателен `X`=deviceId (`AccountId`→`Data.user_id`, lower/UUID; не-UUID → `ignored/invalid_account_id`) + гейт `Status=="Completed"`&`OperationType=="Payment"` (ci → `ignored/not_a_completed_payment`); `TransactionId`/`product_id` — опц. контекст лога.
- **Резолв deviceId→userId** ([ADR-053](../../adr/ADR-053-cloudpayments-webhook-user-resolution-via-auth-devices.md), без изменений): users→auth_devices→`ignored/user_not_found` (**без** исходящего GET).
- **Верификация — единственный триггер начисления:** `GET {CLOUDPAYMENTS_API_BASE}/users/{deviceId}/payments` (`Bearer CLOUDPAYMENTS_API_TOKEN`, 15с). `api_error` (таймаут/не-2xx/malformed) → **`500` retriable** (broadapps перешлёт; НЕ начисляем). 2xx → **реконсиляция** `select_creditable_payments`: `status ∈ CLOUDPAYMENTS_PAID_STATUSES` (дефолт `{succeeded}`) в окне `CLOUDPAYMENTS_PAYMENT_FRESHNESS_HOURS` (дефолт 72ч).
- **Начисление по каждому свежему `succeeded`-платежу** (своя транзакция, идемпотентно `cp-txn:{payment_id}`): класс по `product.payment_type` (`one_time`→tokens \| `subscription`→subscription \| иное→пропуск `unknown_payment_type`); сумма по `product.code` из серверных карт (`TOKEN_PRODUCTS`/`CLOUDPAYMENTS_PRODUCT_TOKENS`+fallback; not-in-map → пропуск `unknown_product`). Дедуп по broadapps `payment_id` (колонка `cloudpayments_webhook_events.transaction_id`:=`payment_id`, репурпозинг без миграции, [Q-054-3](../../99-open-questions.md)).
- Исходы: `applied` (`creditedCount≥1`) / `duplicate` (все дубли) / `ignored/no_creditable_payment` (нет свежих succeeded, WARNING). Ответ `200 {"code":0}` кроме `429`/`500`.
- Наблюдаемость: `cloudpayments_webhook_outcome` + `verify`/`creditedCount`/`paymentStatuses`/`resolvedVia`. `classify_product` в начислении **не используется** (по `payment_type`); остаётся для checkout.
- **НЕ трогать:** миграции (нет), anti-tamper, резолв ([ADR-053](../../adr/ADR-053-cloudpayments-webhook-user-resolution-via-auth-devices.md)), checkout ([ADR-051](../../adr/ADR-051-cloudpayments-checkout-payment-link.md)) кроме переиспользования конфигов/httpx.

**Общее (сохраняется):**
- **PII:** карт-данные (`CardFirstSix`/`CardLastFour`/`Issuer`/`CardType`), `CLOUDPAYMENTS_API_TOKEN`/bearer, `customer_email`, тело ответа verify **не логируются и не персистятся**; `payload` таблицы/audit — только санитизированная проекция.
- Audit `cloudpayments_payment` через `assert_no_secrets`. SSRF-защита: deviceId — валидированный UUID в path, хост из `CLOUDPAYMENTS_API_BASE`.
- **Swagger-чистота** ([R2ter](../../08-api-documentation.md)): user-facing OpenAPI-строки без ADR/TD/Q и внутреннего жаргона.

**Исторически ([ADR-050](../../adr/ADR-050-cloudpayments-webhook.md)/[ADR-052](../../adr/ADR-052-cloudpayments-webhook-lenient-auth-header.md), отменено [ADR-054](../../adr/ADR-054-cloudpayments-webhook-payment-verification.md)):** статический bearer `CLOUDPAYMENTS_WEBHOOK_TOKEN` (constant-time, `401` на mismatch, `500` если не задан) + терпимый разбор заголовка; классификация паттерном имени; идемпотентность по callback `TransactionId`. Заменено публичным эндпоинтом + верификацией.
