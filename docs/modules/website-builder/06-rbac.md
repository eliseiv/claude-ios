# Website Builder — RBAC

## Роль
- `user` — владеет своими проектами и файлами сайта. Единственная пользовательская роль ([05-security.md](../../05-security.md)).

## Tool-доступ (`site.*`)
- Server-side tools исполняются в рамках аутентифицированного chat-шага: `userId` берётся из сессии (`sub` JWT, сверен Gateway),
  `external_project_id` — из `chat_sessions.project_id` сессии. Модель **не** передаёт `userId`/`projectId` в args → не может
  адресовать чужой проект.
- `projects.user_id` всегда = владелец сессии. `site_files` доступны только через проект владельца.

## Preview-доступ
- `GET /v1/preview/*` авторизуется **signed URL** (HMAC + TTL), а не пользовательским JWT ([ADR-010](../../adr/ADR-010-backend-hosted-preview.md)).
- Владелец «запечён» в подпись (`ownerUserId`); сверяется с `projects.user_id`. Доступ к чужому проекту невозможен
  (подделка ломает HMAC). Истёкший/невалидный токен → `403`; чужой/несуществующий проект/файл → `404`.
- Превью не участвует в пользовательской сессии (нет cookies/credentials).

## Изоляция
- Нет admin-доступа к чужим проектам в этом проходе (admin-модуль ограничен начислением кредитов, [modules/admin](../admin/README.md)).
- Эскалация между пользователями невозможна: tool-контекст из сессии + signed URL под конкретного владельца.
