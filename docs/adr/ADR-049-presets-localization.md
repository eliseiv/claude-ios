# ADR-049 — Локализация пресетов промтов: per-instance дефолт + резолвинг локали для `GET /v1/presets`

- **Статус:** Accepted
- **Дата:** 2026-07-02
- **Связано:** [ADR-035](ADR-035-prompt-presets-endpoint.md) (статический реестр пресетов + `GET /v1/presets` — расширяется этим ADR), [Q-035-2](../99-open-questions.md) (локализация `title`/`prompt` — **частично закрывается**), [ADR-034](ADR-034-user-model-selection.md) (паттерн per-provider allowlist в `config.py` — образец конфиг-валидации), [ADR-033](ADR-033-llm-provider-abstraction.md) (провайдер-агностичность), [ADR-017](ADR-017-shared-server-traefik-deploy.md) (мульти-инстанс, per-instance env), [ADR-037](ADR-037-chatrunrequest-context-allowlist-injection.md) (`context.locale` — независимый механизм, см. §7)

## Контекст

По [ADR-035](ADR-035-prompt-presets-endpoint.md) каталог пресетов (`src/app/chat/presets.py`) статичен, зашит на английском и идентичен на всех инстансах; i18n был сознательно отложен ([Q-035-2](../99-open-questions.md)). Появилась подтверждённая потребность: русскоязычный инстанс **avelyra** должен отдавать пресеты по-русски, чтобы чипы на главном экране чата читались на языке аудитории инстанса.

Требование узкое и предметное:
- локализуется **только** `GET /v1/presets` (никакие другие эндпоинты/каталоги в scope не входят);
- локализуются **только** отображаемые тексты `title` и `prompt`; `id` и `icon` (SF Symbol) — стабильные машинные идентификаторы, **не переводятся** (иначе сломается клиентская аналитика/кэш по `id` и рендер `Image(systemName:)`);
- нужен per-instance дефолт (avelyra=ru, остальные=en) **и** возможность per-request переопределения (клиент может явно запросить конкретную локаль).

Инварианты:
- **Аддитивно, обратная совместимость полная.** Без env и без запроса локали ответ идентичен текущему (EN) — существующий контракт `GET /v1/presets` не ломается. Без миграции БД (реестр остаётся в коде, [ADR-035 §2](ADR-035-prompt-presets-endpoint.md)).
- **EN — канон и fallback.** Английский набор — источник истины; любая незаполненная локаль/поле деградирует к EN.
- **Расширяемость.** Механизм не должен хардкодить ровно 2 локали — добавление новой локали = дополнение реестра + набора поддерживаемых локалей, без изменения контракта.
- **Провайдер-агностичность ([ADR-033](ADR-033-llm-provider-abstraction.md)) сохраняется:** локаль — это язык текста пресета, она не зависит от LLM-провайдера инстанса.
- **Порядок чипов сохраняется** во всех локалях (порядок объявления в реестре = порядок на экране, [ADR-035 §3](ADR-035-prompt-presets-endpoint.md)).

## Решение

### 1. Локализованный каталог (реестр в коде)

Реестр `src/app/chat/presets.py` расширяется так, чтобы `id` и `icon` **не зависели** от локали, а `title` и `prompt` имели по одному значению на локаль. Структура — расширяемая (набор локалей задаётся данными, а не двумя жёстко зашитыми ветками). Рекомендуемая форма (backend может выбрать эквивалентную, сохранив инварианты):

```python
# Набор поддерживаемых локалей — единый источник истины (EN первым = канон/fallback).
# Расширение = добавить локаль сюда И заполнить title/prompt в реестре. Не хардкодить «ровно 2».
SUPPORTED_PRESET_LOCALES: tuple[str, ...] = ("en", "ru")
DEFAULT_PRESET_LOCALE: str = "en"  # канон и fallback


class Preset(NamedTuple):
    id: str                    # стабилен, НЕ локализуется
    icon: str                  # SF Symbol, стабилен, НЕ локализуется
    title: dict[str, str]      # locale -> title; ключ "en" ОБЯЗАТЕЛЕН (канон)
    prompt: dict[str, str]     # locale -> prompt; ключ "en" ОБЯЗАТЕЛЕН (канон)
```

- `id` и `icon` — одно значение на пресет, общее для всех локалей.
- `title`/`prompt` — словари `locale -> строка`; ключ `"en"` **обязателен** для каждого пресета (канон/fallback).
- Порядок объявления пресетов в реестре — единый для всех локалей (порядок чипов).
- **EN-тексты — без изменений** относительно текущего `_PRESETS` ([ADR-035 §3](ADR-035-prompt-presets-endpoint.md)); RU-тексты — полный каталог ниже (§1.1).

`id`/`icon` — как в EN ([ADR-035 §3](ADR-035-prompt-presets-endpoint.md), не меняются). Ниже — **утверждённые** RU-значения `title`/`prompt` (естественный профессиональный русский; backend заносит их дословно в `title["ru"]`/`prompt["ru"]`, не меняя `id`/`icon`/порядок):

|---|---|---|---|

| `meeting_notes` | `person.2` | Заметки со встречи | Преврати мои черновые заметки со встречи в аккуратное резюме: ключевые решения, задачи с ответственными и открытые вопросы. Я вставлю заметки следующим сообщением. |

Все 7 RU `title`/`prompt` непусты; порядок строк = порядок чипов = declaration order EN-набора. Это **окончательные** тексты (в отличие от [ADR-035 §3](ADR-035-prompt-presets-endpoint.md), где backend мог уточнять формулировку) — заносятся дословно.

### 2. `preset_catalog(locale)` — сборка с per-field fallback

Сигнатура функции реестра расширяется параметром локали:

```python
def preset_catalog(locale: str) -> list[dict[str, Any]]:
    """Каталог пресетов для указанной локали. Неизвестная/незаполненная локаль → EN-fallback
    (по полю), порядок объявления сохраняется. Pure, без I/O (ADR-033/ADR-035 §6)."""
```

- Для каждого пресета: `title = p.title.get(locale) or p.title[DEFAULT_PRESET_LOCALE]`, аналогично `prompt`. **Per-field fallback**: если у локали заполнена лишь часть полей, недостающие берутся из EN (набор всегда полон и консистентен).
- Неизвестная локаль (не в `SUPPORTED_PRESET_LOCALES`, либо ключ отсутствует в словарях) → каждое поле деградирует к EN → фактически EN-каталог.
- Возврат — список `{id, title, icon, prompt}` в порядке объявления (форма элемента не меняется относительно [ADR-035](ADR-035-prompt-presets-endpoint.md)).
- Функция остаётся pure (без I/O), тестируема; резолвинг локали в неё **не** входит (это concern роутера, §3).

### 3. Резолвинг локали (роутер `GET /v1/presets`)

`src/app/api_gateway/routers/presets.py` определяет фактическую локаль по строгому порядку приоритета. Резолвинг выносится в чистый helper (например `resolve_presets_locale(...) -> str`) — тестируемый отдельно:

**Порядок (первый сработавший выигрывает):**
1. **Query `?locale=`** — если параметр присутствует: значение нормализуется (`strip().lower()`) и должно входить в `SUPPORTED_PRESET_LOCALES`. Валидное → берётся оно. **Невалидное (не из набора) → `422`** (см. политику ниже).
2. **Заголовок `Accept-Language`** — если query отсутствует: парсится первый **поддерживаемый** primary-subtag. Значение разбивается по `,`, у каждого элемента отбрасывается `q`-вес (`;q=...`), берётся primary-subtag до `-` и приводится к lower (`ru-RU` → `ru`, `en-US` → `en`); первый subtag, входящий в `SUPPORTED_PRESET_LOCALES`, — результат. Ни одного поддерживаемого / заголовок пуст/нераспознан → переход к шагу 3 (**тихий fallback**, без ошибки).
3. **`settings.presets_default_locale`** (per-instance env, §4) — если предыдущие шаги не дали локали.
4. **`DEFAULT_PRESET_LOCALE` (`"en"`)** — финальный fallback.

**Политика невалидного значения (обоснование):**
- **Явный `?locale=` вне набора → `422`** (`"locale '<x>' is not supported"`). Query-параметр — явное, полностью клиент-контролируемое намерение; молчаливая подмена скрыла бы опечатку клиента (не отдали бы то, что запрошено). Симметрично строгому контракту `422 unsupported_model` ([ADR-034 §3](ADR-034-user-model-selection.md)) — «явно запрошенное недопустимое значение падает громко». `422` — это техническая ошибка валидации запроса, не бизнес-blocked ([ADR-004](ADR-004-blocked-http-200.md) не применяется: `/v1/presets` — не chat-эндпоинт).
- **Невалидный/непонятный `Accept-Language` → тихий fallback** (шаг 3). Заголовок часто выставляется ОС/системой и не является строго контролируемым клиентом намерением; жёсткий `422` на «неудобный» `Accept-Language` ломал бы обычные запросы браузера/системы. Lenient-парсинг (взять первый поддерживаемый, иначе игнорировать) — стандартная семантика content-negotiation.

### 4. Config: `PRESETS_DEFAULT_LOCALE` (per-instance)

`src/app/config.py`:

```python
presets_default_locale: str = Field(default="en", alias="PRESETS_DEFAULT_LOCALE")
```

- Public, не секрет, per-instance ([ADR-017](ADR-017-shared-server-traefik-deploy.md)). Дефолт `"en"` = текущее поведение (обратная совместимость: незаданный env → EN).
- **Валидация значения — graceful (не startup-crash).** Значение нормализуется (`strip().lower()`) и сверяется с `SUPPORTED_PRESET_LOCALES`; если вне набора → трактуется как `DEFAULT_PRESET_LOCALE` (`"en"`) с записью **WARNING** в лог на старте/при чтении. Обоснование: единообразие с уже принятым в проекте паттерном «мис-конфигурация env деградирует к безопасному дефолту, а не роняет процесс» (`token_products()`/`allowed_models_for()` возвращают fallback на кривом JSON, [ADR-034 §1](ADR-034-user-model-selection.md), а не бросают исключение). Инстанс продолжает обслуживать запросы (EN), опечатка видна в логе. Реализовать чтением через метод-хелпер (например `resolved_presets_default_locale()`), а не сырым полем — по образцу `apple_audience_resolved()`/`normalized_service_domain()`.

### 5. Схема ответа: поле `locale`

`src/app/schemas/presets.py`:
- `PresetInfo` — **без изменений полей** (`id`, `title`, `icon`, `prompt`). `title`/`prompt` теперь несут строку выбранной локали (тип не меняется).
- `PresetsResponse` — **добавляется** поле `locale: str` = фактически отданная локаль (та, что выбрал резолвинг §3, всегда из `SUPPORTED_PRESET_LOCALES`). Аддитивно и обратно-совместимо: старые клиенты игнорируют новое поле; новым — полезно (подтверждение, какой язык отдан, для UI/аналитики).

```json
{
  "locale": "ru",
  "presets": [
    { "id": "plan_week", "title": "Планирование недели", "icon": "calendar",
      "prompt": "Помоги спланировать предстоящую неделю. ..." }
  ]
}
```

**Коды:** `200`; `401` нет/невалидный JWT; `422` явный `?locale=` вне набора; `429` rate-limit.

### 6. Совместимость и инварианты

- **Без env и без запроса локали → EN как сейчас** (полная обратная совместимость с [ADR-035](ADR-035-prompt-presets-endpoint.md)).
- **Без миграции БД** — реестр в коде.
- **`id`/`icon` стабильны** во всех локалях; **порядок чипов** (declaration order) сохраняется во всех локалях.
- **Провайдер-агностичность** сохранена: локаль не зависит от `LLM_PROVIDER`.
- **Read-only, без побочных эффектов**: не создаёт сессию, не пишет ledger/audit, биллинг неизменен ([ADR-006](ADR-006-credit-billing-and-subscription-grant.md)).
- **Rate-limit/auth** — без изменений (`CurrentUser` + `enforce_other_limits`, [ADR-035 §1](ADR-035-prompt-presets-endpoint.md)).

### 7. Отношение к `context.locale` ([ADR-037](ADR-037-chatrunrequest-context-allowlist-injection.md))

`context.locale` в `/v1/chat/run` ([ADR-037](ADR-037-chatrunrequest-context-allowlist-injection.md)) и `?locale=`/`presets_default_locale` в `/v1/presets` — **разные, независимые механизмы**: первый влияет на язык **ответа модели** в конкретном ходе, второй — на язык **статических текстов чипов** каталога пресетов. Они не связаны кодом и не обязаны совпадать (клиент волен показать RU-чипы, но попросить ответ на другом языке). Пересечение только смысловое (оба про язык) — общей реализации не вводим.

### 8. Swagger-чистота (нормативно для backend, [08-api-documentation.md §R2ter](../08-api-documentation.md))

**ВСЕ** user-facing OpenAPI-строки нового/изменённого контракта — **без** `ADR-NNN`/`TD-NNN`/`Q-NNN` и внутреннего жаргона, лаконичный профессиональный русский:
- `Field(description=...)` нового поля `PresetsResponse.locale`;
- описание нового query-параметра `locale` (и его пример/enum-подсказка);
- `summary`/`description` роута `GET /v1/presets` (если правятся — упомянуть параметр `locale` и per-instance язык по умолчанию кратко, без ссылок).

Внутренние ссылки (`ADR-049`, `Q-035-2` и пр.) — **только** в docstring/комментариях кода и в `docs/`.

## Альтернативы

- **Отдельные локализованные эндпоинты (`/v1/presets/ru`).** Отклонено: дублирует контракт, плохо расширяется, ломает единый ресурс. Один эндпоинт + резолвинг локали — стандартнее.
- **Только `Accept-Language`, без `?locale=` и без per-instance дефолта.** Отклонено: не покрывает требование «avelyra отдаёт RU по умолчанию» и не даёт клиенту явно выбрать язык независимо от системного заголовка.
- **`422` и на невалидный `Accept-Language`.** Отклонено: заголовок не строго клиент-контролируем (ОС/браузер), жёсткая ошибка ломала бы штатные запросы; content-negotiation по конвенции — lenient.
- **Startup-crash на невалидном `PRESETS_DEFAULT_LOCALE`.** Отклонено: расходится с принятым в проекте паттерном graceful-деградации env-мис-конфигурации (`token_products`/`allowed_models`); один опечатанный env не должен ронять инстанс. Выбран fallback EN + WARNING.
- **Перевод `id`/`icon`.** Отклонено: `id` — стабильный машинный ключ (аналитика/кэш клиента), `icon` — имя SF Symbol (ресурс iOS); перевод сломал бы обе роли.
- **Хранение локализаций в БД/env-JSON сразу.** Вне scope: реестр остаётся в коде ([ADR-035 §2](ADR-035-prompt-presets-endpoint.md)); путь в config-JSON/БД уже зафиксирован ([TD-026](../100-known-tech-debt.md), [Q-035-1](../99-open-questions.md)) и от локализации не зависит.

## Последствия

- **Положительные:** avelyra (и любой будущий инстанс) отдаёт пресеты на своём языке через `PRESETS_DEFAULT_LOCALE`; клиент может явно запросить локаль (`?locale=`) или положиться на `Accept-Language`; EN — надёжный канон/fallback; механизм расширяется на новые локали без изменения контракта; полностью аддитивно (без env/без запроса — прежнее EN-поведение), без миграции.
- **Цена:** реестр пресетов усложняется (per-locale словари вместо плоских строк) — нужно поддерживать паритет наборов (EN обязателен, прочие локали — по возможности, с per-field fallback); резолвинг локали добавляет helper + тесты.
- **Tech debt / открытое:** i18n остаётся частичным — покрыт **только** `GET /v1/presets` (не другие каталоги); редактирование текстов/локалей без деплоя не реализовано (реестр в коде, [TD-026](../100-known-tech-debt.md)); расширение набора локалей — по потребности. [Q-035-2](../99-open-questions.md) переводится в статус «частично закрыт (пресеты локализуемы, механизм задан); i18n прочих поверхностей — открыто».
- **Безопасность:** тексты пресетов — статичные константы под review (не пользовательский ввод); `?locale=` строго валидируется по замкнутому набору (нет инъекции произвольных ключей); новых секретов/поверхности нет; read-only под общими size/rate/JWT-правилами `/v1/*`.
