# ADR-001 — Выбор стека: Python + FastAPI + PostgreSQL, модульный монолит

- Статус: Accepted
- Дата: 2026-05-21

## Context
Backend для одного iOS-приложения: оркестрация Claude, политика доступа, кредиты, BYOK, tool-loop, аудит. Команда небольшая, нужен быстрый старт, async I/O (внешние API + БД), строгая корректность биллинга. Стек частично зафиксирован пользователем (Python + FastAPI + PostgreSQL).

## Decision
- Язык/framework: **Python 3.12 + FastAPI 0.115** (async).
- БД: **PostgreSQL 16** через **SQLAlchemy 2 async + asyncpg**, миграции **Alembic**.
- Валидация: **Pydantic v2**.
- Вспомогательное хранилище: **Redis 7** (rate limit, idempotency-метки, policy cache).
- Claude: **anthropic Python SDK** с prompt caching.
- Топология: **модульный монолит** — 7 модулей как внутренние пакеты с явными границами, один deployable.
- Точные версии и команды — в [02-tech-stack.md](../02-tech-stack.md).

## Consequences
- (+) Быстрый старт, единая транзакционная БД упрощает атомарность ledger.
- (+) Async хорошо ложится на tool-loop и параллельные внешние вызовы.
- (+) Один deployable — простой деплой, отладка, локальный запуск.
- (−) Модульный монолит требует дисциплины границ (enforced код-ревью), иначе расползание.
- (−) Вертикальное масштабирование на старте; горизонтальное — репликами stateless API.

## Alternatives
- **Микросервисы (7 сервисов)** — отвергнуто: преждевременная сложность, распределённые транзакции для ledger, оверхед для команды на старте.
- **Node.js / Go** — отвергнуто: стек зафиксирован пользователем (Python/FastAPI).
- **Синхронный Flask/Django** — отвергнуто: tool-loop и параллельные внешние вызовы выигрывают от async.
