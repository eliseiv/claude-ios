## Q-015-1 — продуктовое противоречие монетизации MVP (Closed)

**Статус:** Closed — 2026-06-02, решение пользователя. **Выбран вариант (б): покупка токенов требует активной подписки** (докупка сверх месячного пакета).

**Выявлено qa+reviewer (2026-06-02)** при синхронизации реализованного token-purchase с policy: при прежнем дефолте «покупка разрешена всем» пользователь без подписки мог купить кредиты, но не мог их потратить (потребление в `credits`-mode гейтится [ADR-002](adr/ADR-002-access-policy-state-machine.md): без подписки доступен только trial) — «мёртвый» баланс.

**Принятое решение (вариант б).** Покупка токенов осмысленна только при активной подписке: `POST /v1/tokens/purchase` гейтится policy-guard'ом — `subscription.status == active` проверяется **до** `WalletService.grant`. Без активной подписки → HTTP `403` с `code=subscription_required`, кредиты **не** начисляются, ledger не пишется. Покупка = докупка кредитов **сверх** `SUBSCRIPTION_CREDITS_PER_PERIOD` для подписчиков. Это **сохраняет** §2 ТЗ и [ADR-002](adr/ADR-002-access-policy-state-machine.md) без изменений (policy-engine и правило потребления не меняются), устраняет «мёртвый» баланс. Вариант (а) (policy-патч, разблокировка `credits`-mode без подписки покупкой) — **отвергнут**.

## Q-017-1 / Q-017-2 — deploy-значения домена и Traefik certresolver (Closed)

**Статус:** Closed — 2026-06-02, финальные deploy-значения от пользователя/владельца сервера ([ADR-017](adr/ADR-017-shared-server-traefik-deploy.md)).

**Q-017-1 (домен).** `SERVICE_DOMAIN=broadnova.shop`, публичный URL `https://broadnova.shop`. Предзапусковое операционное требование (не код): A-запись `broadnova.shop` → `87.239.135.154` должна существовать **до** запуска — она нужна внешнему Traefik для ACME HTTP-challenge при выпуске Let's Encrypt-сертификата. Остаётся пунктом [prod-checklist](07-deployment.md#prod-readiness-checklist-must-configure-before-launch).

**Q-017-2 (certresolver).** `TRAEFIK_CERTRESOLVER=le` (`--certificatesresolvers.le.acme...` в конфиге общего Traefik `/opt/edge`). Владелец сервера сделал `le` **дефолтным** резолвером на entrypoint `websecure` (`--entrypoints.websecure.http.tls.certresolver=le`), поэтому сертификат выпускается автоматически для любого HTTPS-роутера, и label `traefik.http.routers.<service>.tls.certresolver=le` в нашем `docker-compose` **опционален**. Тем не менее явный label **рекомендован для надёжности** (детерминированность при возможном изменении дефолта на entrypoint).
